- A user of this application must be able to access the app from their browser on mobile and desktop.
- They should be provisioned a way to set up the Github Oauth app (via environment variables).

## Regular Flow
- Login with Github
- List and filter repositories (search)
- Clone a repository into a workspace
- List workspaces
- Select a workspace
- Delete a workspace
- When a workspace is selected, create a new terminal for that workspace or retrieve all the saved processes
- Fetch history of an active terminal
- Allow user to switch between single, dual - vertical and horizontal, tripple, quard pane layouts
- Each layout can have one or more terminal tabs
- See status of the processes/system resource usage etc
- List files of current selected workspace
- Preview a selected files
- Create/delete a file
- Edit content of a selected file with syntax highlighting
- Choose and remember a theme
- Refresh authentication if stale (to allow to fetch new credentials and be able to pull and push to github)
- Allow configuration of multiple users who can log in via multiple github usernames in the environment
- Allow a user to logout, which clears and also removes Oauth application's access to the user's github a/c.
